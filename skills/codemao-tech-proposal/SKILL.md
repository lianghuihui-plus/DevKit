# CodeMao 技术方案生成器

## 概述

根据需求文档（PRD、规格说明等），按 `templates/移动端技术方案模版.md` 生成结构化的 CodeMao 移动端技术方案文档。

生成后如需修改内容，直接在对话中告知即可，无需走修订流程。

## 何时使用

- 用户提供了需求文档，要求输出 CodeMao 技术方案
- 用户说「写技术方案」「出技术设计文档」
- 不要用于：纯代码实现、单元测试编写、PRD 撰写

## 配置

- 模板文件：`templates/移动端技术方案模版.md`
- 若模板文件不存在，直接结束流程，告知用户「模板文件缺失，请将模板放到 templates/ 目录下」

## 工作流程

### Step 0：交互式收集输入

逐一向用户询问，收到回答后再问下一个：

1. 先问：**「请提供需求文档的路径（多个文件或目录用逗号分隔）」**。收到后逐一校验：
   - 若是文件 → 校验存在且可读
   - 若是目录 → 自动将该目录下**一级**的文档类型文件（`.md`、`.doc`、`.docx`、`.xls`、`.xlsx`、`.pdf`、`.txt` 等）纳入输入；忽略子目录和非文档文件
   - 有缺失则指出并重新询问
2. 再问：**「技术方案输出到哪个目录？（留空默认当前目录）」**。
3. 最后问：**「文件名是什么？（如 `xxx-技术方案.md`，留空默认 `技术方案.md`）」**。

完成后跳至 Step 1。

### Step 1：读取模板与输入文件

1. 确认 `templates/移动端技术方案模版.md` 存在。不存在则告知用户并结束。
2. 读取模板全文，理解各章节结构、占位说明和格式要求。模板开头的引用说明块（`> 技术方案加入编辑人和审阅人的目的：...`）**保留**到输出中。
3. 逐一读取全部输入文件，提取关键信息：功能点、约束条件、接口依赖、数据实体、边界场景等。
4. 将提取的信息按模板章节分类整理，以便 Step 2 逐节填充时直接取用，避免回头反复翻原文。

完成后跳至 Step 2。

### Step 2：逐节生成方案

按模板的章节顺序填充。模板章节结构如下：

| 章节 | 填充规则 |
|------|----------|
| 开头引用说明 | 保留模板原文不动 |
| 功能版本表 | 从输入提取版本信息；撰写人、审阅人、状态留空 |
| `# 1、需求概述` | 从输入提取；1.1 需求背景（模板中的需求文档链接、交互UI链接、后端方案链接三个字段，输入中有则填入，无则留空）；1.2 需求范围 |
| `# 2、技术架构` | 2.1 需求架构图（如输入有架构信息则绘制 mermaid 图，否则标注 `【待补充】`）；2.2 技术选型（列出技术栈） |
| `# 3、功能模块设计` | 根据需求拆分为 3.1、3.2… 逐模块描述功能、技术实现、数据结构 |
| `# 4、边界特殊场景` | 按模板中的编辑器类型（KN/Python/Nemo/Kids）逐一检查是否涉及；若需求未涉及某编辑器，保留标题并标注「不涉及」；APP前后台切换、IM音视频冲突、平板适配、登录态变化按需填充 |
| `# 5、影响范围` | 列出改动影响的功能模块和测试关注点 |
| `# 6、接口对接` | 从输入提取接口定义，按模板格式填入；无接口则标注「无」 |
| `# 7、前端对接` | 从输入提取前端协议/桥接定义，按模板格式填入；无则标注「无」 |
| `# 8、需求疑问` | 从输入提取待澄清的需求点；无则标注「无」 |
| `# 9、风险同步` | 评估改动风险（如影响主流程、需灰度开关等）；无则标注「无」 |

填充原则：
- 输入有信息 → 如实填入
- 输入无信息 → 标注 `【待补充】`
- 可选章节（6-9）无内容 → 标注「无」
- 模板中预置的链接类字段（如需求文档链接、交互UI链接等），无内容时留空
- 绝不编造信息

多文件冲突处理：
- 若不同输入文件对同一事项描述矛盾，标注冲突并优先采用用户最后指定的文件中的信息
- 格式：`（注：文档A描述为X，文档B描述为Y，此处采用文档B）`

完成后跳至 Step 3。

### Step 3：输出并校验

1. 将生成的完整方案写入输出文件
2. 执行输出前自检
3. 告知用户输出路径

## 输出前自检

- [ ] 模板的 9 个一级章节全部覆盖
- [ ] 二级章节（1.1、1.2、2.1、2.2、3.1~3.N、4.1~4.7）全部覆盖
- [ ] `【待补充】` 仅用于输入未覆盖的信息，未编造内容
- [ ] 功能版本表格式与模板一致
- [ ] 输出文件可正常打开且格式完整

## 常见问题

1. **编造信息。** 输入文件没覆盖的，标 `【待补充】`，不要猜。
2. **删除模板章节。** 即使是可选章节（6-9）也要保留标题，无内容标注「无」。
3. **遗漏边界场景。** 模板中 4.1~4.7 的编辑器/场景必须逐一过，不要跳过。
4. **覆盖已有文件。** Step 0 若输出文件已存在，提醒用户确认。
5. **忘了读模板就生成。** 务必在 Step 1 先读取模板全文，再在 Step 2 按模板结构逐节填充。
