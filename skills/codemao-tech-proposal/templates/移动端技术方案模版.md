# 移动端技术方案模版

> 技术方案加入编辑人和审阅人的目的：

1. 质量保障
   Leader/核心开发经验丰富，能发现技术盲点和架构缺陷，确保方案的技术深度和实施可行性。

2. 风险控制
   提前识别技术风险、实施风险、业务风险，避免重大技术决策失误。

3. 责任明确
   编辑人：负责方案完整性和准确性，审阅人：负责技术合理性和风险评估，建立可追溯的责任体系。

4. 知识传承
   形成团队统一的技术标准，提升团队整体技术能力。

5. 决策优化。
   多视角评估，避免个人决策局限性，通过充分讨论找到最优技术路径。

## 7、前端对接【可选】

协议接口

```objc
- (NSString *)get_app_info:(NSString *)msg {
  NSString *version = [[[NSBundle mainBundle] infoDictionary] objectForKey:@"CFBundleShortVersionString"];
  NSString * phoneSystemVersion = [[UIDevice currentDevice] systemVersion];
  NSDictionary *params = @{
    @"user_id" : NotNilStr(CodeMaoAccount.defaultManager.userInfoModel.user_id),
    @"sys" : @"ios",
    @"sys_ver" : NotNilStr(phoneSystemVersion),
    @"app_pid": CMLConfig.sharedConfig.pid,
    @"app_ver" : NotNilStr(version),
    @"user_code" : NotNilStr(CMLAppConfig.sharedConfig.appConfigModel.userCode)
  };
  return [params mj_JSONString];
}
```
